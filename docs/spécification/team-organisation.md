# Documentation sur l'organisation du groupe
L'organisation du groupe est un livrable à part entière. Vous devez démontrer que votre équipe a fonctionné comme une équipe technique réelle, pas comme quatre personnes qui codent chacune dans leur coin.

## Conditions
• Chaque membre du groupe doit avoir une responsabilité principale identifiable (ex. : lead BDD, lead back-end, lead front-end, lead conception/UX), sans pour autant travailler en silo.
• Un tableau de répartition figure obligatoirement dans le dossier (qui a fait quoi, en heures estimées et en livrables produits).
• Un seul nom dominant sur le dépôt = sanction sur la note de groupe et possibilité de notes différenciées.

## Proposition
Chacun doit avoir un rôle principal identifiable pour le livrable final. 
*Même si tout le monde touche à tout, c'est cette personne qui est responsable devant le groupe de sa section.*

*   **[Nom 1] (Lead Tech / Architecture)** : Setup CI/CD, Docker, validation des PRs, intégration globale.
*   **[Nom 2] (Lead Data / Backend)** : Modélisation SQL/NoSQL (MCD/MLD), création des scripts de seed, logique métier API.
*   **[Nom 3] (Lead Frontend / UI)** : Intégration React, responsive design, appels API.
*   **[Nom 4] (Lead Produit / QA)** : Rédaction du dossier (Sections 5 à 11)[cite: 1], création des wireframes/maquettes, tests de l'API (Postman).
